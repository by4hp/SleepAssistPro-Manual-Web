# SleepAssistPro Manual Web

## 项目定位

- 这是 `SleepAssistPro` 的用户手册网页项目，不是产品 PRD，也不是应用源码仓库。
- 主要内容来源是 `lib/docs-data.ts` 中的正文数据，以及 `public/manual/**` 下的截图资源。
- 输出默认使用中文，页面里用户实际会看到的英文按钮名、页面名和字段名保留英文原文。

## 手册写作要求

- 面向用户写，不面向研发或内部团队写。
- 直接对用户说话，优先使用“您”。
- 一句话只说一个动作，先写用户现在在哪，再写下一步做什么。
- 优先写用户看得到的页面名、按钮名、开关名和弹窗名。
- 语气保持平实、可靠、克制，让用户读一遍就知道下一步做什么。
- 优先优化文字流畅性和流程合理性，不追求堆信息。

## 强约束

- 不要根据截图自行猜测未确认的产品行为。
- 不要变更已确认步骤的顺序，除非用户明确要求改流程。
- 不要改变步骤与图片、流程图之间的对应关系。
- 不要随意改动截图路径、图片顺序和锚点 id。
- 如果文案、截图和现有结构存在冲突，优先保留已确认流程，并把不确定内容标成 `待确认`。

## 结构约束

- 章节说明先交代“这章适合谁看”或“这章能帮您完成什么”。
- 连续操作使用编号步骤。
- 补充说明使用短段落或短列表，不写长串并列句。
- 二级目录正文里的内嵌小标题使用三级标题语义，不要继续降成更小层级。

## 第五章改写规则

- 第五章是系统设置章节，重点是把入口、操作动作、结果说明写清楚。
- 可以优化语句顺序、段落节奏和提示语，但不要改变现有步骤与配图的先后关系。
- 对高风险操作，例如 `Disconnect Bed`、诊断、退出登录，先让用户知道结果，再写操作。
